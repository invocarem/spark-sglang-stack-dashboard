Spark Sglang Stack Dashboard

About this dashboard

Starter picks a stack preset and a script under scripts/sglang/*.sh (or vLLM). Start runs the stack and the script in one server request. Optional vLLM testing uses a model id for Chat/Benchmark; other tabs: Logs, Tools, metrics, benchmark, chat.

Starter — SGLang

Choose a stack preset and a script (first option Stack only = idle container, no inference server). Use Refresh status first (Docker container for the preset, container list, inference server), then Start — creates/refreshes the stack with PID 1 sleep infinity, then runs SGLang from your .sh via docker exec so the container stays up if the server exits. Output goes to /workspace/.monitor/sglang-launch.log (see Logs). Refresh status updates both lines below. Stop SGLang stops inference only (container keeps running). Stop container runs docker stop on the stack preset container (with --rm it is removed). Use Advanced to target a different container for the server line and stops.

How it works

Host repo ./scripts/sglang mounts at /workspace/scripts/sglang. The API With a stack preset, Start runs docker run … sleep infinity (PID 1 keeps the stack alive), writes the exact run to .monitor/monitor-stack-<preset>.docker-run.sh, then docker exec -d bash /workspace/.monitor/monitor-launch-<script>.rendered.sh (replay helper: .monitor/monitor-launch-<script>.docker-exec.sh; model flags in .rendered.body.sh). Advanced: pick a container only — same exec path into an existing idle stack. Optional HF_TOKEN is passed into the container; mount root is MONITOR_REPO_ROOT (or auto-detected). Open Logs to tail the launch log; SGLang metrics show readiness.

Launch args parsed from script (python3 -m sglang.launch_server or sglang serve): edit values or uncheck to remove unsupported flags.

Multi-node cluster

Optional export lines merged before the server command: NCCL / GLOO / MASTER_* (same for SGLang and vLLM—PyTorch / NCCL pick them up). Use different interfaces on each host; run Start per node. Repo .env can prefill via the API when you use npm run dev. SGLang: merges --dist-init-addr / --nnodes / --node-rank into launch_server when missing from the script. vLLM: merges --master-addr / --master-port (from the MASTER_ADDR / MASTER_PORT fields), --nnodes, and --node-rank. Set --distributed-executor-backend (mp vs ray) in your script—see vLLM serve CLI (--dist-init-addr is SGLang-only).

Advanced — target container

Override which container is used for the Inference server line and for Stop SGLang / Stop container (default is the stack preset’s name when it appears in docker ps).
